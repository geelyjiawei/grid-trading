## 1. 项目概览

本项目是一个个人自用的**合约网格自动交易网站**，功能参考币安合约网格交易。  
用户在浏览器中配置网格参数后，后端自动在交易所挂单、监听成交、循环补单，实现价格震荡区间内的自动套利。

**当前支持：** Bybit（合约永续，线性合约 USDT 结算）  
**规划支持：** Binance（合约永续）

### 核心功能

- 做多网格（Long Grid）：低买高卖，价格上涨方向盈利
- 做空网格（Short Grid）：高卖低买，价格下跌方向盈利
- 实时行情展示（价格、涨跌幅、24h 成交量）
- 账户余额与持仓实时展示
- 挂单列表与成交记录
- 累计利润统计

## 5. 核心模块说明

### 5.1 `bybit_client.py` — 交易所 API 封装

负责所有与 Bybit 交易所的通信。使用 HMAC-SHA256 签名算法。

**主要方法：**

| 方法 | 类型 | 说明 |
|------|------|------|
| `get_ticker(symbol)` | 公开 | 获取实时行情（价格、涨跌幅） |
| `get_instrument_info(symbol)` | 公开 | 获取合约精度信息（tickSize、qtyStep） |
| `get_balance()` | 签名 | 获取 UNIFIED 账户 USDT 余额 |
| `set_leverage(symbol, leverage)` | 签名 | 设置杠杆倍数（买卖方向同时设置） |
| `place_order(...)` | 签名 | 下限价单（GTC，可指定 reduce_only） |
| `cancel_order(symbol, order_id)` | 签名 | 撤销单个订单 |
| `cancel_all_orders(symbol)` | 签名 | 撤销该合约所有挂单 |
| `get_open_orders(symbol)` | 签名 | 获取当前所有挂单 |
| `get_positions(symbol)` | 签名 | 获取当前持仓 |
| `get_order_history(symbol)` | 签名 | 获取历史订单 |
| `round_to_step(value, step)` | 工具 | 按交易所精度四舍五入（向下取整） |

**签名流程：**

```
param_str = timestamp + api_key + recv_window + payload
signature = HMAC-SHA256(api_secret, param_str)
```

### 5.2 `grid_engine.py` — 网格策略引擎

网格策略的核心大脑，负责：

1. **初始化**：从交易所拉取合约精度 → 设置杠杆 → 计算格线 → 挂初始订单
2. **轮询**（每 3 秒）：对比当前挂单与本地记录，检测哪些订单已成交
3. **补单**：成交后按策略在对应格线挂对向单

**关键数据结构：**

```python
# 活跃挂单字典，key = order_link_id
active_orders = {
    "g_3_B_abc123": {
        "link_id":    "g_3_B_abc123",
        "order_id":   "1234567890",     # 交易所返回的真实订单 ID
        "level_idx":  3,                # 第几格（从下往上，0 开始）
        "side":       "Buy",            # "Buy" | "Sell"
        "price":      "95000.0",
        "qty":        "0.01",
        "status":     "open",
        "reduce_only": False            # True = 平仓单，False = 开仓单
    }
}

# 历史成交列表
filled_orders = [
    {
        "side":      "Buy",
        "price":     95000.0,
        "qty":       0.01,
        "level_idx": 3,
        "profit":    10.0,   # 该格利润 = 格线间距 × 数量
        "time":      1714012800.0
    }
]
```

**成交检测原理：**

```
每次轮询时：
  open_order_ids = {交易所当前所有挂单 ID}
  本地 active_orders 中，order_id 不在 open_order_ids 里的 → 视为已成交
```

> 注意：这种轮询方式简单可靠，但有 3 秒延迟。生产环境建议改用 WebSocket 监听成交推送。

### 5.3 `main.py` — FastAPI 主程序

- 提供 RESTful API，供前端 `app.js` 调用
- 维护两个全局变量：`_client`（BybitClient 实例）和 `_engine`（GridEngine 实例）
- 使用 `asyncio.create_task` 让网格引擎在后台异步运行，不阻塞 API 响应
- 静态文件（`frontend/`）由 `StaticFiles` 托管，`/` 路由返回 `index.html`

## 6. 网格策略逻辑

### 6.1 格线计算

```
价格区间 [lower, upper]，分成 N 格
格线列表 = [lower, lower+step, lower+2*step, ..., upper]
共 N+1 个格线，N 个格子
step = (upper - lower) / N
```

示例：BTC，区间 90000 ~ 100000，10 格：
```
格线：90000, 91000, 92000, ..., 100000
格子：[90000,91000], [91000,92000], ..., [99000,100000]
```

### 6.2 做多网格（Long Grid）

**适用场景：** 预期价格在区间内震荡，整体偏涨

| 情况 | 操作 |
|------|------|
| 买单成交（低价买入开多） | → 在上一格价格挂卖单（平多/止盈） |
| 卖单成交（高价卖出平多） | → 在下一格价格挂买单（重新开多） |

初始挂单规则：
- 当前价格 **以下** 的格线 → 挂买单（开多，`reduce_only=False`）
- 当前价格 **以上** 的格线 → 挂卖单（平多，`reduce_only=True`）

```
格线:  90k  91k  92k  [当前93k]  94k  95k
挂单:  买   买   买              卖   卖
```

### 6.3 做空网格（Short Grid）

**适用场景：** 预期价格在区间内震荡，整体偏跌

| 情况 | 操作 |
|------|------|
| 卖单成交（高价卖出开空） | → 在下一格价格挂买单（平空/止盈） |
| 买单成交（低价买入平空） | → 在上一格价格挂卖单（重新开空） |

初始挂单规则：
- 当前价格 **以上** 的格线 → 挂卖单（开空，`reduce_only=False`）
- 当前价格 **以下** 的格线 → 挂买单（平空，`reduce_only=True`）

```
格线:  90k  91k  [当前92k]  93k  94k  95k
挂单:  买   买              卖   卖   卖
```

### 6.4 每格利润估算

```
每格利润（USDT）= 格线间距 × 每格合约数量
格线间距        = (价格上限 - 价格下限) / 网格数量
所需保证金估算  = (价格下限 × 每格数量 × 网格数量) / 杠杆
```

## 9. Bybit API 对接

### 9.1 端点

| 环境 | Base URL |
|------|---------|
| 正式网 | `https://api.bybit.com` |
| 测试网 | `https://api-testnet.bybit.com` |

### 9.2 签名规则（V5）

```python
timestamp  = str(int(time.time() * 1000))
recv_window = "5000"

# GET 请求：payload = query string
# POST 请求：payload = JSON body string
param_str  = timestamp + api_key + recv_window + payload
signature  = hmac.new(api_secret.encode(), param_str.encode(), sha256).hexdigest()
```

**请求头：**
```
X-BAPI-API-KEY:      <api_key>
X-BAPI-SIGN:         <signature>
X-BAPI-SIGN-TYPE:    2
X-BAPI-TIMESTAMP:    <timestamp>
X-BAPI-RECV-WINDOW:  5000
Content-Type:        application/json
```

### 9.3 合约精度处理

Bybit 对每个交易对规定了价格和数量的最小步长，下单必须严格遵守，否则报错。

```python
# 精度信息来源
GET /v5/market/instruments-info?category=linear&symbol=BTCUSDT

# 关键字段
priceFilter.tickSize    # 价格最小步长，如 "0.1"
lotSizeFilter.qtyStep   # 数量最小步长，如 "0.001"
lotSizeFilter.minOrderQty  # 最小下单数量，如 "0.001"

# 处理方式（向下取整）
formatted_price = floor(price / tickSize) * tickSize
formatted_qty   = max(qty, minOrderQty)
formatted_qty   = floor(qty / qtyStep) * qtyStep
```

### 9.4 下单参数说明

```json
{
  "category":    "linear",      // 线性合约（USDT 结算）
  "symbol":      "BTCUSDT",
  "side":        "Buy",          // "Buy" | "Sell"
  "orderType":   "Limit",        // 限价单
  "qty":         "0.01",
  "price":       "95000.0",
  "timeInForce": "GTC",          // Good Till Cancel，挂单直到成交或撤销
  "reduceOnly":  false,          // true = 只允许减仓（平仓单）
  "orderLinkId": "g_3_B_abc123"  // 自定义订单 ID，便于本地追踪
}
```

## 10. 后续开发计划（Binance）

### 10.1 新增文件


### 10.2 Binance Futures API 差异

| 项目 | Bybit V5 | Binance Futures |
|------|---------|----------------|
| Base URL | `api.bybit.com` | `fapi.binance.com` |
| 签名算法 | HMAC-SHA256，签名拼接方式不同 | HMAC-SHA256，签名附加在 query string |
| 签名位置 | 请求头 | Query string `&signature=xxx` |
| 合约类型 | `category=linear` | 路径前缀 `/fapi/` |
| 最小数量字段 | `lotSizeFilter.qtyStep` | `LOT_SIZE` filter 的 `stepSize` |
| 杠杆接口 | `POST /v5/position/set-leverage` | `POST /fapi/v1/leverage` |
| 下单接口 | `POST /v5/order/create` | `POST /fapi/v1/order` |
| 持仓接口 | `GET /v5/position/list` | `GET /fapi/v2/positionRisk` |

### 10.3 改造思路

1. 在 `main.py` 增加 `exchange` 参数（`"bybit"` 或 `"binance"`）
2. `_get_client()` 根据 `exchange` 返回对应客户端实例
3. `GridEngine` 本身不需要改动，只依赖客户端接口

### 10.4 前端改造

- 导航栏增加交易所切换按钮（Bybit / Binance）
- API 配置弹窗增加 `exchange` 选择
- 切换交易所时重置状态，重新拉取配置

## 11. 常见问题

**Q: 启动报错 `ModuleNotFoundError`**  
A: 依赖未安装，在项目根目录执行 `pip install -r requirements.txt`

**Q: API 验证失败 `invalid api_key`**  
A: 检查 API Key 和 Secret 是否复制完整，注意前后不要有空格；确认是否选错了正式网/测试网

**Q: 下单报错 `Order quantity has too many decimals`**  
A: `qty_per_grid` 填写的精度超过了交易所允许的步长，参考 Bybit 合约详情页确认最小数量

**Q: 价格区间内没有任何挂单**  
A: 当前价格不在设定的 `lower_price` ~ `upper_price` 区间内，价格超出范围网格不会挂单

**Q: 停止网格后持仓没有平仓**  
A: 停止网格只撤销所有**挂单**，不会自动平仓（与币安行为一致）。需要手动在 Bybit 网页端平仓，或后续添加"停止并一键平仓"按钮

**Q: 服务器重启后网格状态丢失**  
A: 当前状态存在内存中，重启即清空。如需持久化，可将 `GridEngine` 的状态序列化到本地 JSON 文件或 SQLite 数据库
